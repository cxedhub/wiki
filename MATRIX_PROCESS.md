# Adding a Lesson to the Cross-Disciplinary Matrix

The matrix is derived from two sources of truth:

1. `content/lessons/*.md` — per-lesson YAML frontmatter (the lesson itself).
2. `data/standards.json` — description + source URL for every standard code.

Everything else — the `const L=[…]` in the matrix HTML, the `const S={…}`
tooltip lookup, the prose "379 lessons · 5,072 codes" counts, and
`static/js/standards-db.js` — is generated by `scripts/rebuild_matrix.py`.
**Never hand-edit those files.**

## Add or update a lesson in three steps

### 1. Create / update the lesson markdown

```yaml
---
title: My New Lesson
description: One sentence students would find useful.
date: '2026-04-19T10:00:00'
draft: false
author: First Last
subjects:
  - Computer Science
  - Mathematics          # or any other subject(s)
grade_levels:
  - 6-8                  # pick from K-2, 3-5, 6-8, 9-12
cs_domains:
  - Algorithms and Programming
cs_principles:
  - Creating Computational Artifacts
standards:
  CSTA:                  # ≤ 8 codes
    - 2-AP-10
    - 2-AP-17
  ISTE:                  # ≤ 6 codes
    - ISTE-4a
    - ISTE-5c
  NGSS:                  # ≤ 5 codes
    - MS-ETS1-1
  CCSS Math:             # ≤ 5 codes (omit the framework entirely if unused)
    - 7.EE
standard_types:          # must mirror the keys above, alphabetised
  - CCSS Math
  - CSTA
  - ISTE
  - NGSS
materials: Website, micro:bit
tags: []
---
```

Rules of thumb:

- Every code under `standards:` must also appear in `standard_types:`
  (otherwise taxonomy pages and filters miss the lesson).
- Only list codes the lesson **actually addresses** — the per-framework caps
  above exist so the matrix stays useful, not exhaustive.
- Do not invent new codes. Use ones that already exist in
  `data/standards.json`. If you need a code that isn't there, add it in
  step 2 first.

### 2. If you referenced a standard that isn't in the database yet

Open `data/standards.json`, find the right framework block, and add one
alphabetically-sorted entry:

```json
"3A-IC-26": "Demonstrate ways a given algorithm applies to problems across disciplines",
```

The `descriptions` block is a flat `code → description` map per framework.
The rebuild script resolves URLs automatically from `framework_urls` plus
per-code rules (NGSS, CCSS patterns) — you don't need to write URLs by hand.

### 3. Regenerate everything derived

```bash
python3 scripts/rebuild_matrix.py
```

The script prints a summary like:

```
Lessons discovered:   380
Standards alignments: 5082
Unique codes used:    268

Updated:
  static/matrix/index.html
  static/matrix/app.html
  content/_index.md
  ...
```

If you see a warning about "code(s) used in lessons but missing from
data/standards.json", go back to step 2 — the warning lists the exact codes
and the lesson that first referenced each.

Commit the lesson markdown **plus** whatever the rebuild touched. If you
forget, CI's `--check` mode will fail (see below).

## Verifying in CI or pre-commit

`python3 scripts/rebuild_matrix.py --check` exits non-zero if any derived
file is out of date. Wire it into pre-commit or CI to catch drift:

```yaml
# Example: .github/workflows/matrix-check.yml
- run: python3 scripts/rebuild_matrix.py --check
```

## Updating the tracking snapshot

`MATRIX_TRACKING.md` holds a human-readable snapshot of what the matrix
currently covers. Regenerate its numbers whenever you add a meaningful
batch of lessons:

```bash
python3 scripts/rebuild_matrix.py      # keeps derived files in sync
# then hand-update the tables in MATRIX_TRACKING.md with the new stats
# (the script prints everything you need)
```

## Why this exists

Before this process, the matrix HTML was hand-edited and the prose counts
drifted (the page advertised "417 lessons" long after the corpus had been
cleaned to 379). Eighteen standard codes were used by lessons but had no
description or source link because `standards.json` hadn't kept up with
new alignments. Both problems went unnoticed because there was no
single command to rebuild from source. This document and
`scripts/rebuild_matrix.py` close that loop.
